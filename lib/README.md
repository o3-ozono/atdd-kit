# lib/

Shell library scripts shared across skills, hooks, and the BATS test suite.

## Files

| File | Purpose |
|------|---------|
| `scenario_loader.sh` | Validate and echo a headless skill-chain scenario spec (JSON). See `docs/guides/headless-skill-testing.md`. |
| `skill_assertion.sh` | Subsequence / strict / forbidden match engine for skill-chain assertions. |
| `skill_fix_dispatch.sh` | Dispatch, inflight registry, env scrubbing, and cleanup for the `skill-fix` background subagent flow. |
| `skill_transcript_parser.sh` | Extract Skill `tool_use` events from a stream-json transcript. |
| `spec_check.sh` | Single source of truth for spec-file detection and slug derivation (US/AC spec files). |
| `autopilot_convergence.sh` | Convergence safety rails for autopilot (`autopilot`, #246): normalized-fingerprint sameness-detector and stuck detection using **same-step FAIL rows only** (PASS rows are never part of the comparison population — #277), max-iterations, and the JSONL audit log. Includes `record_halt` for appending a terminating HALT record on convergence-failure halts (#299) with enum `MAX_ITERATIONS / sameness-detector / stuck / ac-drift / log-integrity / gate-unverifiable` (#355 F2), and `record_iteration` timestamps each audit line (ISO 8601 UTC). A corruption guard (#248) refuses (non-zero → halt) when a FAIL row's fingerprint was partial-written / corrupted, instead of silently dropping it (fail-OPEN). Also includes the **#334 deterministic red gate**: `record_red_evidence <red-jsonl> <test-commit-sha> <at-file> [impl-sha]` appends a JSONL line recording that the AT was observed RED at the test commit; the optional `impl-sha` 4th argument (#355 F8) records the impl baseline SHA alongside the test SHA so `check_red_evidence` can read it from the record instead of reconstructing via git log; `check_red_evidence <test-commit-sha> <impl-commit-sha> <red-jsonl> [git-dir]` returns 0 only when both SHAs are present, red evidence exists for the test commit, and the test commit is a strict git ancestor of the impl commit — fail-closed (non-zero) on any missing/invalid input, absent evidence, or order violation. Pure bash + coreutils. |
| `lease-store.sh` | Generic capacity-1/key cross-session lease for `full-autopilot` (#318): `issue` pool (one claim per Issue) and `merge` pool (`main-merge`, serializes integration). Acquire is **atomic (`mkdir` lock) + fail-closed** — if the holder cannot be persisted the lock is released and acquire fails, never reporting a non-persisted lease as held. TTL orphan cleanup uses the holder.json timestamp, falling back to the lock-dir mtime so a freshly-created lock is not mis-reclaimed mid-race. Pure bash + coreutils. |
| `merge-coordinator.sh` | merge coordinator failure state machine for `full-autopilot` (#318): records a failure and decides retry vs escalate at threshold N (auto-retry → human escalation), plus a `process` orchestration `rebase → re-gate → merge → regression` whose external steps are env-injectable (`MC_REBASE_CMD` / `MC_REGATE_CMD` / `MC_MERGE_CMD` / `MC_REGRESSION_CMD`). post-merge regression failure surfaces as `merged:regression-failed` (non-zero), never swallowed. Pure bash + coreutils. |
| `full-autopilot-dispatch.sh` | dispatcher select/release for `full-autopilot` (#318): given K slots and a queue of candidate Issues, `select` returns up to K for which an `issue` lease can be acquired (skipping Issues claimed elsewhere); `release` frees an issue-lease when a worker completes. Composes `lease-store.sh`. GitHub-state prefilter (`is_issue_busy`) skips Issues with an open PR or `in-progress` label before attempting lease acquisition (#326); the busy-check is env-injectable via `FAD_BUSY_CMD` so callers can stub GitHub API calls without network access. The parallel-exclusion / 数珠つなぎ core. |
| `full-autopilot-run.sh` | `full-autopilot` dispatcher runtime (#318, #329): the queue-drain loop — **dynamic** `$FA_QUEUE_CMD` re-evaluation on every slot-refill (US-1 / #329: replaces the old startup-freeze; new issues added to `ready-to-go` mid-session are picked up in the same run) → select (issue-lease) → launch headless worker → monitor (bash-3.2-portable `kill -0` poll, no `wait -n`) → merge-ready ⇒ merge coordinator under merge-lease → release issue-lease → refill slots and re-evaluate queue. Dedup across in-flight, lease-held, and completed sets prevents double-dispatch. **Notify preflight (US-2):** at `run()` start, logs a warning if `FA_NOTIFY_CMD` is unset, or a confirmation line if set; does not stop. **merge-ready double-check (US-3):** `__default_result` checks `is_error:false` self-report **AND** `gh issue view --json labels` for a `merge-ready` GitHub label — label absent → `failed`; both present → `merge-ready`. All external steps (queue / launch / result / merge / notify) are env-injectable (`FA_QUEUE_CMD` / `FA_LAUNCH_CMD` / `FA_RESULT_CMD` / `FA_MERGE_CMD` / `FA_NOTIFY_CMD`); default launcher runs `FA_HANDOFF=1 CLAUDE_CODE_PRINT_BG_WAIT_CEILING_MS=0 claude -p "/atdd-kit:autopilot <i> --hand-off"` in a per-Issue **worktree** (the ceiling env lifts headless `claude -p`'s 600s background-wait cap so the worker's design/impl Workflow runs to completion instead of being terminated mid-loop) (default `<repo-parent>/<repo-name>-<i>`, override `FA_WORKTREE_DIR`, `FA_NO_WORKTREE=1` for cwd launch), **seeding the gitignored `.claude/settings.local.json` (`enabledPlugins` + `extraKnownMarketplaces`) into it first (#329)** — unseeded the headless worker cannot resolve the plugin and dies with `Unknown command: /atdd-kit:autopilot`; default merge wires real git steps (`fa-merge-steps.sh`). Fires `FA_NOTIFY_CMD <event> <issue> <detail>` on dispatch / merge-ready / merged / merge-failed / worker-failed / escalate (`FA_NOTIFY_LEVEL` quiet|normal|verbose). `FA_WORKER_TIMEOUT` kills a hung worker tree; trap releases in-flight leases on shutdown; lease store is repo-scoped + session unique. |
| `fa-merge-steps.sh` | Real git merge steps for the merge coordinator (#318): `rebase <branch>` (onto origin/main→main, abort on conflict) and `merge <branch>` (`--no-ff` into base, abort on conflict), operating on `FA_REPO`. Wired into `__default_merge` via `MC_REBASE_CMD`/`MC_MERGE_CMD` so the production merge path runs real git (gh-free → throwaway-repo testable, never touches the real main). |

## Usage

Each script prints its own usage when run without arguments, e.g.:

```bash
bash lib/spec_check.sh derive_slug <issue-number>
bash lib/skill_fix_dispatch.sh query_inflight [<skill> <phase>]
```
