# Hiro — Solo Developer

> Primary persona for atdd-kit. Grounded in the repository's actual user pattern: a single maintainer who uses atdd-kit on their own projects to run an Issue-driven, AI-assisted workflow end-to-end.
> See [persona-guide.md](../methodology/persona-guide.md) for field definitions and creation guidelines.

## Name

Hiro Sawaki

## Role / Job Title

Solo developer working on personal projects and side-projects. Uses AI coding assistants (Claude Code) as the primary pair-programmer. No team, no PO, no dedicated reviewer — one person filling every role.

## Goals

### Primary Goal

Ship small, reliable increments on personal projects without burning out on context-switching between planning, implementation, review, and release.

### Secondary Goal

Keep a consistent, self-enforced process so that work started today can be safely resumed weeks later, even after long context gaps.

## Context

| Dimension | Detail |
|-----------|--------|
| Technical level | Expert developer, intermediate at running AI-assisted agentic workflows. Comfortable with git, CLI, GitHub, bash. |
| Environment | macOS, terminal-centric, `gh` CLI, Claude Code desktop + CLI. Works across several personal repos under a personal GitHub account. Frequently switches between machines and idle weeks. |
| Constraints | No teammates to catch mistakes — tooling has to provide the guardrails. Limited after-hours time, so each session has to pick up cleanly from the last. Refuses runtime dependencies that would make setup fragile across machines. |

## Quote

> "I don't need a bigger AI — I need an AI that won't let me skip the steps I'd skip on a tired Tuesday night."
