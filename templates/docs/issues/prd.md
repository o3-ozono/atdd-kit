     品質規律 4 原則（Marty Cagan『Inspired』を基礎とし、atdd-kit 独自
     の追加項目には [独自] を明示する）:
     1. 事実と課題の分離 — 「現状どうなっているか（観察された事実）」と
        「それが何を引き起こすか（課題・影響）」を別欄に書く（Inspired）。
     2. 1 PRD = 1（本質）課題 [独自] — 複数の本質課題があるなら PRD を分ける。
     3. 観察可能なゴール — ゴールは利用者側で観察できる変化として書く。
        内部完了条件（「〜を実装する」「〜ファイルを書く」）は不可（Inspired）。
     4. 下流からの還流 [独自] — 設計・AT・実装で判明した制約を PRD へ
        戻す双方向接続を明示する。

     anti-pattern 集（このいずれかに当てはまったら書き直す）:
     - 事実と課題の混在（例:「〇〇が不便だ」→ 事実か課題か不明）
     - 複数課題の同居（例:「A の問題と B の問題を解消する」を 1 PRD に詰める）
     - 内部完了条件のゴール（例:「テンプレートを書き換える」）
     - 観察不可能な成功指標（例:「品質が上がる」）
     ============================================================ -->
